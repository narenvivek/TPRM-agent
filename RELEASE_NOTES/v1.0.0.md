# Release Notes - v1.0.0

**Release Date**: 2024-11-20
**Type**: Initial Release

## Summary

TPRM Agent v1.0.0 is the initial release of our Third-Party Risk Management platform. This release provides a complete foundation for vendor risk assessment with document analysis, AI-powered risk scoring, and Airtable integration.

## New Features

### Document Upload & Management
- **Multi-file upload**: Upload multiple documents (PDF, DOCX, TXT) simultaneously
- **Document storage**: Files stored with unique URLs for easy access
- **Vendor-based organization**: Documents organized by vendor ID
- **File serving**: Download uploaded documents via secure URLs

### AI-Powered Risk Analysis
- **Gemini 2.5 Flash integration**: Real AI analysis using Google's latest model
- **Structured output**: Risk scores (0-100), risk levels (Low/Medium/High)
- **Findings extraction**: Automated identification of security concerns
- **Recommendations**: AI-generated remediation suggestions
- **Individual document analysis**: Analyze each document separately

### Vendor Management
- **Vendor onboarding**: Create vendors with criticality, spend, and data sensitivity
- **Risk tracking**: Monitor vendor risk scores over time
- **Comprehensive profiles**: Track website, description, and business context

### Airtable Integration
- **Dual-table architecture**: Vendors and Documents tables
- **Automatic sync**: All data synchronized with Airtable in real-time
- **Linked records**: Documents linked to parent vendor records
- **Analysis history**: Track analysis status and results

### Modern UI
- **Next.js 16**: Latest React framework with App Router
- **Dark theme**: Professional dark UI with blue-violet gradients
- **Real-time updates**: Loading states and status indicators
- **Responsive design**: Works on desktop and mobile

## Architecture

### Backend (FastAPI)
- Async/await for non-blocking operations
- Service-based architecture for separation of concerns
- Dependency injection for testability
- Comprehensive error handling with graceful degradation

### Frontend (Next.js)
- App Router for modern React development
- TypeScript for type safety
- Tailwind CSS v4 for styling
- Radix UI components for accessibility

### Services
- `AirtableService` - Vendor and document data management
- `AIService` - Gemini AI integration
- `StorageService` - File storage abstraction (local/cloud)
- `DocumentService` - Text extraction from documents
- `DocumentAirtableService` - Document metadata tracking

## API Endpoints

### Health & Status
- `GET /` - Root health check
- `GET /health` - Detailed health status

### Vendors
- `GET /vendors` - List all vendors
- `POST /vendors` - Create new vendor

### Documents
- `POST /vendors/{vendor_id}/documents/upload` - Upload documents
- `GET /vendors/{vendor_id}/documents` - Get vendor documents
- `POST /documents/{document_id}/analyze` - Analyze document
- `GET /files/{vendor_id}/{filename}` - Download document

## Configuration

### Environment Variables (Backend)
```bash
AIRTABLE_API_KEY=your-airtable-pat
AIRTABLE_BASE_ID=your-base-id
GEMINI_API_KEY=your-gemini-key
STORAGE_TYPE=local
STORAGE_PATH=./uploads
BASE_URL=http://localhost:8000
```

### Environment Variables (Frontend)
```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## Deployment

### Local Development

**Backend**:
```bash
cd backend
uv venv
source .venv/bin/activate
uv pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**Frontend**:
```bash
cd frontend
npm install
npm run dev
```

## Airtable Setup

### Vendors Table Fields
- Name (Single line text)
- Website (URL)
- Description (Long text)
- Criticality (Single select: Low, Medium, High, Critical)
- Spend (Number)
- Data Sensitivity (Single select: Public, Internal, Confidential, Restricted)
- Risk Score (Number)
- Risk Level (Single select: Low, Medium, High)
- Last Assessed (Date)

### Documents Table Fields
- Filename (Single line text)
- Vendor (Link to Vendors table)
- File Type (Single select: pdf, docx, txt)
- Document Type (Single line text)
- File Size (Number)
- File URL (URL)
- Upload Date (Date)
- Analysis Status (Single select: Not Analyzed, Analyzing, Completed)
- Risk Score (Number)
- Risk Level (Single select: Low, Medium, High)
- Findings (Long text - JSON array)
- Recommendations (Long text - JSON array)
